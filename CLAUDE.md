**Parallel roadmap thrust тАФ LOOM PLAYER (Path B тАФ native desktop media player).** DECIDED 2026-07-01: build a branded, self-contained **Loom Player** тАФ a native desktop app (Electron, or grow the existing **Tauri shell at `apps/desktop`**) wrapping a **native decoder (ffmpeg / libmpv)** so it plays **any local `.mkv`** тАФ 10-bit HEVC/x265, AC3/EAC3/DTS/FLAC/TrueHD audio, ASS softsubs тАФ and renders Loom's full 4-layer learning stack over it. **Why native, not a browser tab / extension:** the target audience is the **anime / fansub-torrent community** (who won't adopt a Netflix/streaming extension no matter how good Netflix's JP subs are тАФ it's not their workflow or ethos), and their files are exactly what browsers CAN'T decode (10-bit HEVC + AC3/EAC3/DTS have no browser decoder, especially on Linux). ## Owner Auth Roadmap

Production rate limits (slowapi 100/min, 2000/day per IP; 5000-char per-request cap) protect the slim API from abuse but also block legitimate high-volume operator use тАФ notably the Step 6 OCR synthetic-data pipeline, which will fan out tens of thousands of romanize/annotate calls. The owner-auth path lets Connor (and only Connor) bypass the limiter without weakening defenses for everyone else. Three additive tiers; A satisfies v1.

**Tier A тАФ pre-shared bypass key (тЬЕ shipped, live).** Secret(s) live in Railway env `LOOM_BYPASS_KEYS` (comma-separated, supports rotation). Visit `loom.nerv-analytic.ai/?owner_key=<secret>` once per device тЖТ `OwnerKeyBootstrap` (`apps/web/components/owner-key-bootstrap.tsx`) stashes it in `localStorage.loom_owner_key` and cleans the URL. Every API call then carries `X-Loom-Auth: <secret>` via the `openapi-fetch` middleware in `apps/web/lib/api/client.ts` (read per-request, so a fresh value takes effect immediately). Server-side, `BypassAwareSlowAPI` (`loom_api/web.py`) wraps `SlowAPIMiddleware` and skips the limiter *entirely* for allow-listed keys (`hmac.compare_digest`, constant-time). A floating "owner mode" pill shows when active. **Reset:** `localStorage.removeItem("loom_owner_key")` or visit `/?owner_key=`. **Rotate:** change `LOOM_BYPASS_KEYS` (invalidates all devices at once) and re-issue. Limitations (acceptable for v1): per-device not per-identity; key briefly appears in URL/history; no per-device revocation.

**Tiers B & C (planned / deferred).** B = Google OAuth identity binding (per-email allow-list + minted session JWT); C = Cloudflare Access network gate. Both are strictly additive over Tier A's `X-Loom-Auth` path. Full design, triggers, and cost in **`ROADMAP.md` тЖТ Owner Auth (Tiers B/C)**.

## Layer Terminology тАФ CRITICAL

**Get this right. Every time. No exceptions.**

| Layer name | Screen position | Content | Variable names |
|------------|----------------|---------|----------------|
| **Bottom** | Lowest on screen | User's **native** language (the language the user speaks, e.g. English for an English speaker) | `native_file`, `native_subs`, `native_text`, `bottom_text`, `native_lang` |
| **Top** | Above Bottom | **Foreign / media** language (the language of the video, e.g. Japanese, Thai, Korean) тАФ this is the "target" of the processing/romanization pipeline | `target_file`, `target_subs`, `target_text`, `top_html`, `target_lang_code` |
| **Romanized** | Above Top | Phonetic transcription of the Top/foreign text (Pinyin, Romaji, etc.) | `romaji_text` |
| **Annotation** | Above individual Top tokens | Per-token readings of the Top/foreign text (furigana, bopomofo, etc.) | via `\pos()` in ASS, ruby in PGS |

- "**Native**" = user's own language. NOT the language native to the media.
- "**Target**" = the foreign language being processed/romanized. It is the "target" of the pipeline, not the user's learning target.
- `content_key = (bottom, top, romaji, preserved)` тАФ bottom is native, top is foreign.
- `_derive_region_keys` тЖТ region 0 = top (foreign), region 1 = bottom (native).
- In `build_output_filename()`: `native_lang` = user's language code, `target_lang` = media language code.

## Key Architectural Decisions

**Four-layer output, two independent pipelines.** `.ass` file has 3 or 4 text layers (Bottom / Top / Romanized / optionally Annotation with `\pos()`), controlled by `include_annotations` param. PGS `.sup` is a separate full-frame bitmap rasterization. PlayResX=1920, PlayResY=1080 set explicitly on all generated `.ass` files. All coordinates and font sizes in 1080-scale.

**`.ass` pipeline** (`generate_ass_file()`): no Playwright dependency. `supports_ass_annotation`: CJK=True, R4/Indic/Hebrew=False тАФ gates `\pos()` annotation generation (non-CJK annotation is PGS-only because the layout math assumes CJK glyph widths).

**PGS pipeline** (`generate_pgs_file()` тЖТ `rasterize_pgs_to_file()`):
- Playwright async API, N-worker parallel pool (`num_workers`, default 1). Reorder heap preserves timestamp order; consumer writes sequentially via `SupWriter`. Memory-bounded streaming write.
- Nested event loop support (Streamlit) via background thread.
- ~50тАУ100ms per screenshot; 300 events тЙИ 15тАУ30s.
- Requires `playwright install chromium`.
- **Union timeline** (`_build_pgs_timeline()`): union of all timing boundaries from native + target tracks. One interval per segment so when only one track changes, epoch system emits a Normal update (only changed region re-encoded). Fixes flicker when tracks have independently-timed line breaks.
- **Concurrent event merging** (`_merge_concurrent_target_events()`): groups target events by identical `(start, end)`. Drops music-only events (тЩк, тЩл) when real dialogue is concurrent; stacks remaining concurrent events with `<br>` / `\N`.
- **Canvas-aware region splitting** (`split_regions(canvas_height=)`): gap midpoint must be in 25%тАУ75% of canvas to allow 2-region split. Prevents subtitle dropout when only top-half content is rendered.
- **Epoch management** (`SupWriter.write(region_content_keys=...)`): Epoch Start (full redraw) / Acquisition Point (every 12 display sets, for seek safety) / Normal (only changed region re-encoded) / Skip (identical content). Reserved palette ranges: obj 0 тЖТ indices 1тАУ127, obj 1 тЖТ 128тАУ254. Fixed windows: top 45%, bottom 25%. Abutting threshold тЙд 50ms. Clears always Epoch Start. `region_content_keys=None` falls back to Epoch Start.

**Annotation infrastructure is language-agnostic.** `get_annotation_func(lang_code)` тЖТ span producer. `build_annotation_html(spans, mode)` with 3 render modes: `"ruby"`, `"interlinear"`, `"inline"`. `annotation_font_ratio`: CJK=0.5, alphabetic=0.4. Adding a new annotated script = new `get_annotation_func()` only.

**Container-agnostic input, MKV output.** ffprobe/ffmpeg accept any container. Output always `.mkv`. Subtitle upload accepts `.srt`, `.ass`, `.ssa`, `.vtt`. `loom_core/video/mkv_handler.py` is the only file that touches ffmpeg.

**MKV mux critical flags:** `-c:s:N ass` re-encodes ASS track (fixes timestamp conversion). `-max_interleave_delta 0` forces strict DTS-order interleaving (fixes subtitle clustering). PTS=0 anchor in `SupWriter`/`write_sup()` prevents ffmpeg timestamp rebasing. `merge_subs_to_mkv()` accepts optional `ass_path` + `sup_path`; `disposition:default` on PGS if both present; `default_audio_index` sets audio default; `keep_existing_subs`/`keep_attachments` for track stripping.

**Output filenames:** `build_output_filename()` тЖТ `{media}.{year}.{native_lang}.{target_lang}[.{annotation}][.{romanization}].{ext}`. Title/year from `get_video_metadata()`.

**No RAM-loading of video** тАФ always local path + ffmpeg subprocess.

**Timing offsets** (`shift_events(subs, offset_ms)` in `loom_core/subs/utils.py`): deep-copies SSAFile, shifts all event start/end by `offset_ms`, clamps to >=0. Applied as `native_offset_ms`/`target_offset_ms` immediately after subtitle load in preview/processing call sites. Streamlit UI uses pending-key indirection (`_pending_top_offset_sec`/`_pending_bottom_offset_sec`) to avoid `StreamlitAPIException` on post-widget state mutation.

**Auto-alignment** (`compute_subtitle_offset(reference_subs, target_subs)`): returns `target_time - reference_time` (positive = reference earlier, shift source-A tracks later). Coarse pass = pairwise-difference histogram (N├ЧM pairs, 100ms bins, `Counter`); fine pass = ┬▒2s around peak in 10ms steps, ┬▒500ms tolerance, midpoint of best plateau. Filters Comment events + `\p` drawings; minimum 5 dialogue events per track.

**Output resolution scaling:** `_PLAYRES_OPTIONS` (480pтАУ2160p) + "Match source". `_scale = target_height / 1080` applied to all style attrs.

## Tripwires тАФ load-bearing, don't relearn these

**Backend (`loom_api` on Railway):**
- **Worker recycling MUST go through gunicorn, not `uvicorn --limit-max-requests`.** railway.json's `restartPolicyType` is `ON_FAILURE` тАФ a process that exits CLEANLY (code 0, which is exactly what uvicorn does when it hits its request limit) is **never restarted** тЖТ silent total outage. The gunicorn master never exits; it respawns workers as `--max-requests` retires them. (Procfile + railway.json `startCommand`, worker class `uvicorn_worker.UvicornWorker` тАФ the `uvicorn-worker` package, NOT the deprecated `uvicorn.workers` path.)
- **The result cache must stay fail-open.** `PostgresResultCache` swallows every DB error internally (log + 30s backoff); routes never see cache exceptions and `get_result_cache()` degrades to `NullResultCache` on any construction failure. Never let a cache/DB problem 500 the batch routes тАФ uncached-but-working is the contract (`loom_api/result_cache.py`).
- **Changing romanizer OUTPUT requires an `ENGINE_VERSIONS` bump** (`loom_core/romanize.py`) or the cache serves the old behavior forever and "why didn't my fix take?" ensues. Refactors that don't change output must NOT bump (needless cold start).

**Extension (`apps/extension`):**
- **NO `backdrop-filter` anywhere.** The pill + panel render over YouTube's continuously-repainting player; `backdrop-filter` re-blurs the underlying pixels every frame тЖТ main-thread saturation тЖТ multi-second input lag. Use a solid `rgba(...)` background with opacity тЙе 0.94. (`components/loom-pill.tsx`, `components/settings-panel.tsx` headers.)
- **The pill MUST NOT depend on `target`/`native` from context.** A compact-mode toggle keyed on caption text re-rendered the pill every dialogue boundary, generating new inline styles that triggered overlapping CSS transitions that never settled. The pill reads only `status` and is wrapped in `React.memo`; compact mode was dropped. **(0.2.1) The active pill now auto-fades after 3s idle** тАФ but perf-safely: it's a single `visible` boolean that flips ONLY on the idleтЖФactive edge (never per dialogue), the `mousemove` handler short-circuits via a `visibleRef` so it never `setState`s per-move, and the fade is an `opacity` transition on the already-promoted compositor layer (GPU, no main-thread paint). Don't reintroduce per-frame state on the pill. (`components/loom-pill.tsx`.)
- **The shadow host MUST be on its own compositor layer.** `transform: translateZ(0); will-change: transform; contain: layout paint style` on `loom-overlay-root` (`injectHostPositioningStyle` in `content.tsx`), else YT's progress-bar tick + control auto-hide cascade through our paint surface on the main thread. Same `translateZ(0)` on the pill button for defense-in-depth.
- **WXT `build` defaults to `chrome-mv3`.** `npm run build` тЖТ `.output/chrome-mv3/`; Firefox testing needs `npm run build:firefox` тЖТ `.output/firefox-mv2/`. If UI changes "aren't appearing" after a reload, check `.output/firefox-mv2/` mtime before hunting a logic bug.
- **Nested-`<ruby>` outer-rt position is inverted on Firefox MV2.** Per-rt `ruby-position` is honoured for flat single-rt rubies but ignored (forced to `over`) for the outer rt of a nested ruby. **In the live overlay the Simplified auxiliary ruby therefore renders ABOVE the reading/annotation line** тАФ the desired, kept behavior (reads better pedagogically). Any prose calling it "Simplified below" is wrong; the language coding is unchanged, only the description. NOTE the settings-panel preview still uses flat single-rt rubies and so VISUALLY shows Simplified below the reading тАФ a known preview-vs-live divergence to reconcile in the UI overhaul (`UI_REVISIONS.md` C-7). (`annotated-text.tsx` header.)
- **react-colorful's runtime CSS auto-inject doesn't reach the shadow root.** It `appendChild`s a `<style>` to `document.head`, which our shadow DOM doesn't inherit. The CSS is vendored verbatim in `settings-panel.tsx::REACT_COLORFUL_CSS` (pinned to 5.6.1; re-extract via the regex-walker noted there when bumping).
- **`tlang=` is intrinsically lock-step.** A `tlang` override makes YouTube emit one MT'd event per source event with identical timing тАФ that's the API, not a regression. Visible only where source-event boundaries don't match English sentence boundaries (Chinese mid-clause splits being canonical). If "tracks are no longer independent" is reported, first check whether the native side is `(auto)`/tlang.
- **(0.2.1) The overlay outline INTENTIONALLY diverges from the web PGS reference.** `caption-overlay.tsx::buildTextShadow` is an **8-direction** text-shadow ring (was corners-only, which looked chunky/"chrome"), and outline + drop-shadow + glow alpha are **multiplied by the layer's master opacity** so lowering a line's opacity fades the whole stack together (the fill used to fade alone, leaving the outline floating). This deliberately does NOT match `apps/web/lib/raster/build-html.ts::textShadowCss` тАФ the web path rasterizes to PGS and needs pixel parity with the desktop; this live-HTML overlay does not. Don't "resync" them. (Comment in `caption-overlay.tsx`.)
- **(0.2.1) Dev/prod postMessage channel is namespaced тАФ and MUST stay that way.** `MAIN_SOURCE`/`ISO_SOURCE` (`lib/env.ts`) get a `-dev` suffix when `IS_DEV`. A dev build is routinely loaded side-by-side with the prod "Loom" for live testing; both inject content scripts into the SAME page, which shares one `window`, so a `window.postMessage` from either build's MAIN is seen by BOTH builds' ISO listeners. With a shared tag, the installed prod build's (old-code) MAIN leaked tracklists into the dev ISO and silently overrode the dev build's behavior тАФ this masqueraded for two cycles as "the Netflix fix isn't working." If a dev-build behavior change "has no effect," FIRST confirm the prod build isn't also installed/enabled.

**Extension тАФ Netflix (`apps/extension`, step 5h):**
- **Anchor the overlay to `div[data-uia="player"]`, NOT `div[data-uia="video-canvas"]`.** video-canvas is a child of player; Netflix's control chrome (back/flag/bottom bar) lives in `player`, ABOVE video-canvas's stacking context тАФ so an overlay mounted in video-canvas (even at max z-index) is trapped *below* the controls and the pill is unclickable while the chrome is up. `player` is the lowest common ancestor of the `<video>` and the controls (confirmed via LCA probe), so our max-z wins there. It also wraps video-canvas as the same box (captions/scale unchanged) and is inside the fullscreen subtree. (`lib/overlay/netflix-player-anchor.ts`.)
- **Pill/panel clicks must `stopPropagation`.** Netflix toggles play/pause on a bubble-phase `click` on the player surface (and dblclickтЖТfullscreen). Since the overlay lives inside `player`, every pill toggle / in-menu click leaked to it. Swallowed at the pill + panel boundary via `lib/overlay/stop-player-events.ts` (stopPropagation, not preventDefault). Safe because the panel's click-outside dismiss is a CAPTURE-phase document listener (fires first). YouTube doesn't pause on overlay clicks тЖТ harmless there.
- **Identify tracks by `id`, never `languageCode`.** Netflix exposes multiple tracks per language (plain "English" + "English (CC)"); matching on languageCode collapsed them тЖТ dual-highlight, duplicate React keys, events-cache collisions. `CaptionTrack.id` (YT vssId / NF manifest trackId) is the identity for picker key + selection + cache. `isCc` (`rawTrackType==="closedcaptions"`) lets `auto-pick` prefer standard subtitles while keeping CC selectable.
- **Manifest hook runs MAIN-world at `document_start`.** The hook must monkey-patch `JSON.parse`/`JSON.stringify` BEFORE the player fetches its (MSL-encrypted) manifest. The stringify hook injects the `webvtt-lssdh-ios8` profile so the server returns WebVTT URLs. (`entrypoints/netflix-main.content.ts`.)
- **(0.2.1) "Which episode is playing" is anchored to the MEDIA element, NOT the URL or manifest arrival.** ~21 min into an episode Netflix prefetches the NEXT episode's manifest to buffer ahead AND moves `location` to the next `/watch/<id>` (or briefly blanks it) тАФ all while the `<video>` still plays the current episode's credits. So neither "a new manifest arrived" nor "movieId matches the URL" means the episode changed (a URL-based gate was tried and FAILED for exactly this reason тАФ don't re-attempt it). A manifest for a DIFFERENT title than the one playing is HELD as `pending` and adopted only when the `<video>` fires `loadstart`/`emptied` (the real stream swap). The manifest is always parsed before its media loadstart, so `pending` holds the right episode by swap time. Decision logic is the PURE reducer `lib/captions/netflix/manifest-tracker.ts` (`reduceManifest`/`reduceMediaSwap`, 11 unit tests); `netflix-main.content.ts` just feeds it events + does the postMessage/logging. The capture-phase `loadstart`/`emptied` listener is filtered to the `[data-uia="player"]` video; `durationchange` is logged-only (can fire mid-stream under MSE). Rich `[Loom NFLX MAIN] manifest: тАж` / `video тАж` devlogs reconstruct the sequence offline.
- **(0.2.1) Top defaults to the title's AUDIO language on Netflix.** `serializeTrack` stamps `CaptionTrack.audioLangCode` from the manifest's audio-track array (read defensively тАФ `audio_tracks`/`audioTracks`, prefer a native/original-flagged track else the first); `auto-pick.ts::pickTarget` prefers a foreign track whose base lang matches it OVER tier order. Verified live picking ja on Frieren and the spoken language on ko/zh titles. Undefined `audioLangCode` (e.g. YouTube, or a manifest with no recognizable audio array) тЖТ falls back to tier ordering, no regression.

**Extension тАФ Prime + positional layout (`apps/extension`, 2026-07):**
- **Prime mounts via a STATE-BASED reconciler, NOT `autoMount`.** Prime's player goes 0├Ч0тЖТfull-size via a LAYOUT change, not a DOM mutation, so `autoMount`'s mutation observer never re-fires and the overlay stays stuck on the 0├Ч0 preview. `ensureMount` (`entrypoints/prime.content.tsx`) reconciles to the resolved surface on mutations + a 1 s backstop, migrating on surface **identity** change. (WeTV uses the same pattern for a different reason тАФ player-subtree rebuilds.)
- **The real Prime surface is the LONGEST-DURATION one, not the largest.** Prime autoplays a ~30 s preview trailer on a full-size surface while the real ~6000 s episode loads on a SEPARATE, near-identically-sized surface тАФ "largest area" can't tell them apart (within ~5%). `resolvePrimePlayerSurface` picks by video `duration` (pause-invariant); the overlay anchor AND the playhead binding both go through it so they can't land on different `<video>`s. (`lib/overlay/prime-player-anchor.ts`.)
- **Per-platform prefs MUST be read from a module-level cache, not React state alone.** Prime's reconciler rebuilds the `CaptionProvider` 2тАУ3├Ч on a closeтЖТreopen (previewтЖТepisode migration); a fresh mount at defaults racing the async `storage.local.get` made per-platform size/position settings *appear* not to persist. The prefs live in module-scope `cachedSizeByPlatform` / `cachedPositionByPlatform`, read synchronously in the `useState` initializers so every rebuild restores instantly (`components/caption-context.tsx`). Storage stays the durable source of truth. Corollary: any NEW persisted key must be added to BOTH the setter AND the load-time `storage.local.get([...])` list тАФ position keys were once written but not read, so they silently never round-tripped.
- **Positional / vertical extras render through `zoneAnchor`, kept SEPARATE from the layout function (`zoneStyle`).** `pickPrimary` (`stream.ts`) chooses the horizontal-bottom cue for the main slot and returns null if none exists (a lone vertical cue must NOT be promoted into the horizontal slot + romaji line). Concurrent non-default-zone cues render via `PositionalCue` at their own zone + writing-mode (furigana only). This separation is load-bearing: the per-platform vertical-nudge sliders adjust `zoneStyle` (main lines only) and MUST NOT move signs / vertical cues off their source position. `CueLayout` only ever comes from a DEFINED region/cue-setting (Prime TTML regions `цикф╕Л`/`ч╕жхП│`; Netflix `line:`/`vertical:`), never inferred.
- **`usePlayerScale` measures the visible VIDEO PICTURE, not the player root.** Font sizes scale by `pictureHeight/1080` where picture height = the `<video>`'s `object-fit: contain` box (`min(clientH, clientW┬╖intrinsicH/intrinsicW)`). Measuring the player container instead is why Netflix/YT/WeTV/iQIYI rendered oversized vs Prime (whose anchor already IS the video box). Don't revert it to `playerRoot.clientHeight`. (`lib/overlay/player-scale.ts`.)

**Web app (`apps/web`) тАФ ffmpeg.wasm:**
- The `FFmpeg` class has no `worker.onerror` listener: a worker that fails to boot hangs `load()` forever. Hardened via `FFmpegClient.#init` тЖТ `withTimeout` + window-level error capture on the test page.
- `classWorkerURL` MUST be a fully-qualified URL with origin (`${window.location.origin}/ffmpeg`); a path-only string resolves against `import.meta.url` тЖТ `file://` and the browser blocks it.
- ffmpeg-core MUST be the ESM build (`@ffmpeg/core/dist/esm/`); the module worker does `(await import(coreURL)).default`. UMD has no default export and silently hangs `load()`.
- TS `moduleResolution: bundler` doesn't map `.js` тЖТ `.ts` for value imports тАФ drop the `.js` suffix on imports across `apps/web/lib/`.
- **General rule** (`feedback_async_hang_prevention.md`): every promise from third-party code goes through `withTimeout()` with a labeled rejection. Silent hangs are a banned bug class.

**Desktop (`apps/desktop`):**
- **Dev-mode fonts:** Tauri 2's `resource_dir()` in dev returns the build-artifact dir, not `src-tauri/resources/`; during `npm run tauri dev` set `LOOM_FONT_DIR=$PWD/apps/desktop/src-tauri/resources/fonts` manually. Production bundles read the real resource dir.

## Language Pipelines

Implementation lives in `loom_core/romanize.py` + `loom_core/language.py`. Read those for details тАФ this section captures non-obvious gotchas only.

**Japanese:** `_make_japanese_pipeline()` returns `(resolve_spans, spans_to_romaji)` with closure state (`_romaji_meta` carries merge_mask + particle_ha across calls). fugashi (MeCab) + unidic-lite. Three-tier furigana: author inline `kanji(hiragana)` тЖТ pre-existing ASS furigana тЖТ MeCab fallback. Three long vowel modes: macrons (default) / doubled / unmarked. POS-aware verb chain merging via `_should_merge_for_romaji()`. Particle уБп тЖТ wa via `pos1=хКйшйЮ, pos2=ф┐ВхКйшйЮ`.

**Chinese:** Three variants тАФ `zh-Hans/zh-CN/chs/zh` тЖТ Pinyin, `zh-Hant/zh-TW/cht` тЖТ Zhuyin, `yue/zh-yue/CantoCaptions` тЖТ Jyutping. `_make_pinyin_romanizer()` uses `jieba.cut()` for word boundaries; Traditional тЖТ Simplified via OpenCC `t2s` for jieba (Simplified-oriented dict), boundaries mapped back to Traditional for pypinyin. CJK punctuation stripping via `_is_cjk_punct()` filters punctuation-only segments (covers U+3000тАУU+303F, fullwidth U+FF00тАУU+FF65, etc.).

**Korean:** `korean-romanizer` (Revised Romanization). Per-syllable annotation gives base reading per char (lookup aid); the romanization line uses full-word `Romanizer(text)` which captures liaison/tensification/nasalization (reading aid). Two layers, two purposes тАФ by design.

**Cyrillic:** `cyrtranslit`. `_CYRILLIC_LANG_CODES` maps BCP-47 тЖТ cyrtranslit codes (ru, uk/ua, be/by, sr, bg, mk, mn). Ukrainian/Belarusian disambiguation via `_UKRAINIAN_UNIQUE`/`_BELARUSIAN_UNIQUE` frozensets.

**Thai:** `pythainlp`. 3 phonetic systems: `paiboon` (default, with tone diacritics, vowel remapping aeтЖТ╔Ы ueтЖТ╔п), `rtgs` (no tones, ASCII), `ipa`. Hybrid tokenizer `_thai_tokenize()`: `word_tokenize(engine='newmm')` тЖТ `syllable_tokenize()` on tokens >6 Thai chars. **Critical:** `royin` engine deprecated тАФ mangles consonant clusters; all RTGS/Paiboon+ paths use `thai2rom`. Word boundaries via U+2009 thin space. `annotation_default_enabled: False`.

**Indic (R5-2/R5-3):** Six languages via `aksharamukha.transliterate.process(script, 'IAST', text)` тАФ `_INDIC_SCRIPTS = {hi: Devanagari, bn: Bengali, ta: Tamil, te: Telugu, gu: Gujarati, pa: Gurmukhi}`. Aksharamukha preferred over `indic-transliteration`/sanscript because sanscript distorts Tamil ("vaс╣Зakkam" тЖТ "vaс╣Зaghghaс╣Г") by treating it as Sanskrit-subset. Aksharamukha auto-converts danda (ред) and double-danda (рее) to ASCII periods. Per-akshara annotation: `_split_brahmic_aksharas()` accumulates consonant clusters across virama boundaries тАФ runs aksharamukha per-akshara to get correct conjunct readings (Tamil роХрпНроХ тЖТ "kka") that only whole-unit gives. Bengali Khanda Ta (U+09CE) classified as extender, not standalone тАФ acceptable for reading fidelity. `has_phonetic_layer=True`, `supports_ass_annotation=False`.

**Hebrew (R5-4 phase a):** `_make_hebrew_romanizer()` is consonantal transliteration with two heuristics: (1) mater lectionis тАФ ╫Х/╫Щ are consonantal (v/y) at word-start or after vowel-letter, vocalic (o/i) after consonant; (2) default 'a' inserted between consecutive consonants. Strips nikud/cantillation (U+0591тАУU+05C7). Begadkefat (╫С ╫Ы ╫д) defaults to soft form (v/kh/f) since unpointed Modern Hebrew has no dagesh marker. **Documented failure modes:** ╫С╫и╫Х╫Ъ тЖТ varokh not baruch, ╫Ч╫С╫и╫Щ╫Э тЖТ chavarim not chaverim. Tests lock these in so a future nikud/dictionary-based pass shows up as test diff.

**Arabic / Persian / Urdu (R5-4 remaining):** `_make_arabic_romanizer` / `_make_persian_romanizer` / `_make_urdu_romanizer` share `_arabic_script_romanize_word()` walker. Same mater-lectionis rule as Hebrew on ┘И/┘К (vocalic ┼л/─л after consonant, consonantal w/y at word-start or after vowel-letter). Strips tashkil before transliteration (subtitle text rarely carries it). Three phonetic systems per language (Duolingo-to-academic hybrid):
- **Arabic** тАФ `learner` (default; emphatics с╣г с╕Н с╣н с║У с╕е + long ─Б ─л ┼л + ╩┐/╩╛ + digraphs sh/gh/th/dh/kh) / `din` (full DIN 31635: ┼б ─б с╣п с╕П с╕л) / `loose` (ASCII-only, drops emphatic marks + ayn). Definite article ╪з┘Д handles sun-letter assimilation (14 sun letters double the following consonant: ╪з┘Д╪┤┘Е╪│ тЖТ ash-shams; 14 moon letters keep al-: ╪з┘Д┘В┘Е╪▒ тЖТ al-qamar). Final ╪й (t─Б╩╛ marb┼лс╣нa) тЖТ pause-form "a". Alif maks┼лra (┘Й) тЖТ long ─Б.
- **Persian** тАФ `learner` (default) / `dmg` (single-char digraph alternatives ─Н ┼╛ ┼б ─б с╣п с╕П с╕л). Persian-specific letters ┘╛ ┌Ж ┌Ш ┌п. Persian uses Arabic script but ez─Бfe + vowel inventory differ; emphatic marks are typically collapsed (Persian-style) even in the learner default.
- **Urdu** тАФ `learner` (default) / `ala-lc` (scholarly: candrabindu n╠Р for nun-ghunnah, macron ─У for yeh-barree). Layers on Persian + retroflexes ┘╣ ┌И ┌С тЖТ с╣н с╕Н с╣Ы + nun-ghunnah ┌║ + yeh-barree █Т + aspiration marker ┌╛ (heh doachashmee combines with preceding consonant: ╪и┌╛ тЖТ bh, ┘╣┌╛ тЖТ с╣нh).
- **Documented failure modes** locked in tests: unvocalized short vowels guessed as 'a' (yaktub тЖТ yaktab); no sun-letter assimilation outside ╪з┘Д; P─Бkist─Бn тЖТ P─Бkasat─Бn (default-'a' between k-s).

**RTL rendering (R5-4 phase b):** `is_rtl_text(text, threshold=0.4)` classifies as RTL when Hebrew/Arabic/Syriac/NKo/Samaritan/presentation-form codepoints > 40% of non-whitespace non-digit. `_build_fullframe_html(top_rtl, bottom_rtl)` injects `dir="rtl"` on relevant `#top`/`#bottom` divs; `unicode-bidi: isolate` on every `.layer` so directionality can't leak. Romanized never gets `dir="rtl"`. `generate_pgs_file()` derives: `top_rtl` from target `lang_cfg['rtl']` (authoritative), `bottom_rtl` from content scan over native events (covers arbitrary user languages without needing a `native_lang_code` API param). `.ass` path untouched тАФ libass handles bidi internally.

**Universal romanization polish** (`_polish_romaji(text, *, capitalize=True)`): runs at every romanizer factory tail. Three passes тАФ fullwidth CJK punctuation тЖТ ASCII via `_CJK_TO_LATIN_PUNCT` translate table; strip `\s+` before closing punctuation; uppercase line-start + first alpha after `.!?` when `capitalize=True`. **Capitalize disabled** for Cyrillic (cyrtranslit preserves source case) and Thai (no caps convention). Idempotent.

**Language detection** (`_dominant_script()`): script-specific paths тАФ CJK via `_refine_cjk_detection()`, Cyrillic via `_detect_by_script_chars()` unique-char pre-detection тЖТ langdetect fallback, Thai/Indic by script directly, Latin via `_normalize_metadata_lang()` metadata preference over langdetect (fixes Romance language misidentification). Indic scripts mapped 1:1 via `_INDIC_SCRIPT_TO_CODE`. Hebrew detection: `_dominant_script() == 'Hebrew'` тЖТ 'he'. Arabic-script detection: when `_dominant_script() == 'Arabic'`, trust langdetect's raw_code if it's `ar`/`fa`/`ur`; otherwise default to `ar`. (No unique-letter pre-detection like Cyrillic тАФ Persian-only letters ┘╛ ┌Ж ┌Ш ┌п and Urdu-only ┘╣ ┌И ┌С ┌║ █Т ┌╛ exist, but langdetect was found reliable enough that adding override logic was deferred.)

**Font validation (R6b-fonts):** `loom_core/fonts.py` тАФ `FontScanner` walks one or more font directories, indexes every TTF/OTF/TTC face via `fontTools.ttLib.TTFont`, builds `family тЖТ (path, ttc_index)` + per-face cmap maps. Reads `name` records 16/1/4 (typographic family / family / full name, prioritised, Windows-Unicode platform preferred over Mac Roman) plus OS/2 `usWeightClass` so `resolve()` returns Regular weight when multiple weights of the same family are indexed. Mtime-based lazy rebuild; thread-safe. `validate_font(font_name, *, lang_code=None, text=None, scanner=None)` тЖТ `FontValidation` (resolved_path, resolved_family, resolved_index, is_fallback, coverage_ok, missing_chars, warnings). Per-language samples in `_LANG_COVERAGE_SAMPLES` (zh-Hans uses хЫ╜, zh-Hant uses хЬЛ). Module-level `get_default_scanner()` consults `LOOM_FONT_DIR` (`os.pathsep`-separated) then falls back to platform-conventional system font dirs; `set_default_scanner()` for tests / Tauri startup wiring. **`is_fallback=True` semantics in the new backend** = "requested family not in any scanned dir" (the renderer will pick a system / engine fallback at draw time). UI integration deferred.

## Style System (R6a)

Per-layer controls (Bottom, Top, Romanized, Annotation): color, opacity, font size, font family, outline (toggle + thickness + color + opacity), shadow (toggle + distance, default 1.5), glow (radius 1тАУ20, color, `\blur` ASS tag). `_hex_to_ass_color()` / `_ass_color_to_hex()` bridge `#RRGGBB` тЖФ `pysubs2.Color`. ASS alpha inverted: `int((1 - opacity/100) * 255)`.

**Gap CSS:** `annotation_gap` uses `transform: translateY()` (not `margin-bottom` тАФ broken in Chromium ruby layout) in preview/rasterize. ASS path uses `\pos()` Y-coordinate math.

## Style Mapping

`detect_ass_styles()`: two-pass тАФ pattern match is final, not overridable by event count. Priority: (1) `_PRESERVE_PATTERNS` тЖТ preserve, (2) `_EXCLUDE_PATTERNS` тЖТ exclude, (3) literal "Dialogue"/"Default" тЖТ dialogue (`_DIALOGUE_NAME_RE`), (4) 0 events тЖТ exclude, (5) remaining тЖТ most-events = dialogue. OP/ED/song/karaoke patterns are preserved (not excluded).

`_iter_dialogue_events()`: selects layer with most non-drawing events (not highest-numbered). Excludes all non-main layers. Yields ALL events in the main layer including overlapping ones тАФ concurrent merging is downstream.

`has_animation` detection per style. `_strip_animation_tags()` for PGS path strips `\k`, `\t()`, `\move()`, `\fad()`; preserves visual tags. `.ass` path passes all tags through.

`_dedup_preserved_for_pgs()`: groups by style + time overlap + text content (substring match). Keeps lowest non-drawing layer. Prevents garbled karaoke layer overlap in PGS.
